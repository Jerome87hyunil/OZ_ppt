01. 서비스 기획 배경

프로젝트를 시작한 이유

캠프 내에서 정규 과정 이외에도 바이브코딩에 대한 궁금증이 많은 것 같았고, 바이브코딩에 대한 궁금증을 해소하고, 개발자들이 지식과 경험을 자유롭게 공유하는 활발한 커뮤니티 플랫폼을 구축하고자 하였습니다.

핵심 목표

  • 바이브코딩에 대한 궁금증을 해소
  • 사용자 간 지식 공유 및 협업 활성화
  • Q&A를 통한 신속한 문제 해결 지원
  • 바이브코딩 생태계의 최신 정보 전달

02. 문제 상황 분석

정보의 파편화

지식과 질문들이 여러 채널에 흩어져 있어 정보 접근성이 낮고 검색이 어려움

소통의 부재

캠프원 간의 교류와 협업을 위한 공식적인 소통 창구가 부족하여 커뮤니티 활성화에 한계

전문 커뮤니티 부족

한글로된 바이브코딩에 대한 전문 커뮤니티가 부족하여 정보 접근성이 낮고 검색이 어려움

06. 아키텍처

Frontend

  • Framework Next.js 14
  • Language TypeScript
  • Styling Tailwind CSS
  • UI Kit shadcn/ui
  • State React Query, Zustand

Backend

  • Runtime Next.js API Routes
  • Database PostgreSQL
  • Auth NextAuth.js v5
  • Editor Tiptap

Infrastructure

  • Hosting Vercel
  • DB Hosting Supabase / Railway
  • Image Cloudinary / S3
  • Search PostgreSQL FTS

07. 회고 및 커뮤니티 구축 노력

어려웠던 점

  • 스터디 모임 개설: 부트캠프 내 스터디를 개설하여 학습 문화를 조성했으나, 초기 단계라 아직 운영이 미숙합니다. 앞으로의 효율적인 운영 방향을 정하는 것이 과제로 남아있습니다.

발전시킬 내용

  • 디스코드 및 노션 활성화: 개설된 디스코드 채널과 노션 페이지의 참여를 유도하고 유용한 콘텐츠를 지속적으로 확충하여 커뮤니티의 핵심 소통 창구로 발전시키겠습니다.

08. 멘토링 반영 사항

  • 기술 스택 선정 피드백

    SEO의 중요성을 고려하여 Vite 대신 Next.js를 채택하라는 조언을 적극 반영, 검색엔진 최적화 기반 마련

  • 데이터베이스 스키마 설계

    확장성을 고려한 테이블 구조 설계에 대한 멘토링을 통해 사용자, 게시글, 댓글 간의 관계를 효율적으로 정의

  • MVP(최소 기능 제품) 범위 설정

    프로젝트 초기 단계에서는 핵심 기능(게시판, Q&A)에 집중하라는 조언에 따라 개발 우선순위를 명확히 설정
