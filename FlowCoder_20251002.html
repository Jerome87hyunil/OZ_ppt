바이브코딩의 흐름(Flow)을 체험하는 시간입니다.

발표자: Jerome

스터디: FlowCoder

강의 목표 및 개요

시작하며: 바이브코딩의 3가지 핵심 원칙

  • 1. 규칙 설정 (Rule Setting)
  • 2. Task 단위 (Task Units)
  • 3. 항상 검증 (Always Verify)

오늘 우리가 얻어갈 것들

  • Next.js & DaisyUI를 활용한 신속한 프로젝트 환경 구축
  • AI 코딩 어시턴트(Claude)를 단순 코더가 아닌, 아키텍트로 활용하는 방법
  • 기획(PRD)부터 UI, 개발 Task까지 이어지는 체계적인 개발 프로세스 수립
  • '바이브코딩' : 고민과 반복 작업은 AI에게, 우리는 핵심 로직과 큰 그림에만 집중!

전체 프로세스 한눈에 보기

1. 환경 설정 → 2. AI 파트너 설정 → 3. UI 시스템 구축 → 4. AI 기반 기획(PRD) → 5. UI 구현(Wireframe → Demo) → 6. Task 관리 및 개발

Part 1: 프로젝트 초기 설정

Step 2. UI 라이브러리 설치 (Daisy UI)

CSS, 고민 없이 예쁘게

  • 설치: npm i -D daisyui@latest 명령어를 실행합니다.
  • Daisy UI란?

    Tailwind CSS를 더 쉽게 사용할 수 있게 해주는 플러그인 라이브러리입니다. btn, card 등 직관적인 클래스 이름만으로 완성된 UI 컴포넌트를 빠르게 구현할 수 있습니다.

  • (참고) shadcn/ui 역시 훌륭한 대안입니다.
Part 2: AI 코딩 파트너, Claude 설정

Step 3. Claude 실행 및 권한 설정

AI 개발 파트너 실행

  • 선택은 자유: Claude or Codex
  • 기본 실행 명령어: claude, codex
  • 권한 문제 없이 바로 실행: claude --dangerously-skip-permissions, codex --yolo
  • (주의) 이 명령어는 AI가 프로젝트 파일에 자유롭게 접근하고 수정할 수 있도록 허용하므로, 신뢰할 수 있는 환경에서 사용해야 합니다.

  • Claude.md Agents.md 파일:

Step 5-2. 공통 UI 컴포넌트 자동 생성

반복적인 UI 코딩은 이제 그만!

명령어로 UI 생성 요청

프롬프트 예시:

/sc:implement 아래의 CSS 설정값을 globals.css 에 적용해주고 해당 설정값과 Daisy UI 기반으로 공통 UI 컴포넌트 생성하고 demo 페이지를 생성해줘.

공통 컴포넌트에는 다음 종류를 필수로 추가해줘.

- 히어로, 케루셀, 버튼, 뱃지, 헤더, 네비바, 푸터

(복사한 Theme Code 붙여넣기)

결과: globals.css 업데이트, 공통 컴포넌트 파일들, 그리고 컴포넌트를 바로 확인할 수 있는 데모 페이지까지 한번에 생성됩니다.

결론 및 Q&A

'바이브코딩' 워크플로우 요약

  • AI를 단순 코딩 도구가 아닌, 아키텍트이자 페어 프로그래머로 활용합니다.
  • 반복 작업(환경설정, UI 코딩, 문서화)은 AI에게 위임하고, 창업개발자는 비즈니스 로직과 창의적인 문제 해결에 집중합니다.
  • Claude.md를 통해 규칙 기반의 체계적이고 예측 가능한 개발 프로세스를 구축합니다.

바이브코딩 스터디: 'FlowCoder'?

스터디를 시작하고 싶은 계기: 팀원인 용현님과 서로 가르쳐주고 실행해보는 과정에서 '함께 배울 때 가장 크게 성장한다'는 것을 직접 느꼈습니다. 이는 학습 피라미드 모델에서 말하는 '서로 설명하기'와 같은 원리입니다.

창업개발 부트캠프 수강생 여러분들은 단순한 동료가 아닌, 함께 성장해야 할 소중한 '전우'라고 생각합니다. 각자의 자리에서 고군분투하는 우리가 함께 모여 지식을 공유하고, 서로의 성장을 돕는다면 더 큰 시너지를 낼 수 있을 것이라 확신합니다.

그래서 용기를 내어 'FlowCoder' 스터디를 시작하게 되었습니다. 이 스터디가 꾸준히 이어져, 우리 모두가 AI 시대의 흐름(Flow)을 주도하는 코더(Coder)로 함께 성장해나가는 발판이 되기를 진심으로 바랍니다.
